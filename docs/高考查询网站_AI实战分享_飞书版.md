## 一句话先说结论

如果你是文科生、不会写代码、也没有系统学过编程，依然可以借助 AI 做出一个真的能使用的小网站。

我这次做的是一个：

**高考录取分数线查询网站**

它目前已经可以完成这些基础功能：

- 选择省份
- 选择年份
- 输入分数
- 搜索学校
- 查看学校、专业、最低分、最低位次、批次等信息

它还不是一个完整商业产品，但它已经完成了从**一个想法**到**一个网页工具**的转变。

对我来说，这次最大的收获不是“我学会了编程”，而是：

**原来普通人也可以把自己的需求，真的做成一个工具。**

---

## 为什么想做这个网站？

高考志愿填报最麻烦的地方，不是没有资料，而是：

**资料太多、太散、太难对比。**

一个学生或者家长，经常会反复查这些问题：

- 我的分数能上哪些学校？
- 某个学校去年最低录取分是多少？
- 同一所学校不同专业之间差多少分？
- 今年填报时，哪些学校适合冲，哪些比较稳？

如果每次都在不同网站之间来回切，会非常耗时间，也很容易漏掉信息。

所以我想做一个简单工具：

**把这些数据集中放在一个页面里，通过筛选和搜索，快速查到结果。**

---

## 我是怎么开始的？

一开始，我并没有对 AI 说“帮我写代码”。

我直接用最普通的人话，把需求讲清楚。

### 我最开始是这样描述需求的

```text
我想做一个高考大学录取分数线查询网站。

我是编程零基础的新手，请你用最简单的方式帮我完成。

网站需要支持：
1. 选择省份
2. 输入高考分数
3. 搜索学校名称
4. 筛选年份
5. 展示学校、专业、最低分、最低位次、批次
6. 页面要适合高考生和家长使用，清楚、可信、不要花哨
7. 底部要写明：数据仅供参考，最终以官方信息为准

请先帮我做一个最小可用版本。
```

这里最关键的一句是：

**请先帮我做一个最小可用版本。**

因为零基础最容易犯的错误，就是一开始想做得太复杂。

但真正有效的方式是：

**先做出一个能跑起来、能查询、能展示结果的第一版。**

---

## 第一版网站做了什么？

第一版并不复杂，只完成最核心的查询流程。

### 页面最基础的几个模块

| 模块 | 作用 |
| --- | --- |
| 页面标题 | 告诉用户这是高考录取分数查询工具 |
| 筛选区域 | 选择省份、年份、输入分数 |
| 搜索框 | 搜学校或专业 |
| 结果表格 | 展示录取分数线 |
| 风险提示 | 说明数据仅供参考 |

### 第一版的目标很简单

**输入条件，查到录取分数线。**

就这一件事。

不是一上来就做“院校分析”“智能推荐”“冲稳保系统”“完整全国数据库”。

先把最基础的一步做通，比什么都重要。

---

## 数据一开始怎么处理？

我没有一开始就接数据库。

而是先用一个最简单的 CSV 表格跑通数据流程。

比如：

```csv
年份,省份,学校,专业,最低分,最低位次,批次
2024,河南,郑州大学,计算机类,598,24500,本科一批
2024,河南,河南大学,法学,575,39000,本科一批
2023,河南,郑州大学,计算机类,590,26000,本科一批
```

对零基础来说，这种方式有两个很大的好处：

1. 不需要先理解数据库
2. 可以先验证“这个产品逻辑到底成不成立”

也就是说，先别急着做复杂系统，先把“查询这件事”做通。

---

## 整个过程中，我主要让 AI 帮我做了什么？

我让 AI 帮我做的事情，主要分成 4 类。

### 1. 帮我生成网页结构

包括：

- 首页长什么样
- 查询栏怎么放
- 表格怎么排
- 查询结果页如何展示

### 2. 帮我写筛选和搜索功能

包括：

- 省份筛选
- 年份筛选
- 分数输入
- 学校关键词搜索
- 专业关键词搜索
- 查询结果排序

### 3. 帮我优化页面样式

包括：

- 页面不要太像学生作业
- 风格要更像正式高考志愿查询平台
- 表格要清楚
- 手机端也要能看

### 4. 帮我排查问题

包括：

- 页面打不开
- 页面空白
- 数据没显示
- 布局太松
- 交互不清楚

## 我发现：Vibe Coding 最关键的，不是“会不会写代码”

而是：

**你能不能把需求讲清楚。**

比如页面不好看，我不会说：

“请优化 CSS。”

我会直接说：

```text
这个页面看起来太像学生作业了。
请你把它改得更像一个正式的高考志愿查询工具。
要求：清爽、可信、表格易读、手机上也能看。
```

如果筛选不好用，我也不会说抽象术语，而是直接说：

```text
现在查询体验不够好。
请帮我优化：
1. 省份改成下拉选择
2. 年份改成下拉选择
3. 分数输入框只能输入数字
4. 没有结果时给出提示
5. 结果按照最低分从高到低排序
```

这种方式特别适合新手。

你不需要会很多技术词汇，只需要做到一件事：

**把你看到的问题，讲得足够具体。**

---

## 这个过程中最容易卡住的地方

我觉得零基础最容易卡在 3 个地方。

### 1. 一开始想太复杂

最容易出现的想法是：

- 我想做智能推荐
- 我想做冲稳保
- 我想做院校分析
- 我想做专业对比
- 我想做完整全国数据库

这些都可以做，但它们不该是第一步。

第一步最重要的是：

**先把最基础的查询跑通。**

### 2. 不知道怎么描述问题

很多新手会在页面报错时自己硬猜。

其实最简单的方法就是：

**直接把报错交给 AI。**

```text
这是页面报错信息：[粘贴报错]
请你用新手能听懂的话解释原因，并直接帮我修改。
```

这通常比自己瞎查快很多。

### 3. 本地能打开，不等于真的做完了

很多人会以为网页在自己电脑上能打开，就算完成了。

其实不是。

如果想让别人也能访问，还需要：

1. 上传代码到 GitHub
2. 部署到 Vercel
3. 拿到一个可以分享的线上网址

到这一步，才算真正意义上的“上线”。

## 做完之后，我最大的感受是什么？

以前我一直以为做网站要先学这些：

- HTML
- CSS
- JavaScript
- 数据库
- 部署
- 前后端开发

但这次实践让我意识到：

对普通人来说，也可以反过来。

**先借助 AI 做出一个能用的东西，再慢慢理解它。**

这条路径的好处是：

你不会卡死在“还没学会，所以不能开始”。

相反，你会先看到成果，再倒推理解。

这件事对我最大的改变，不是“我突然成为程序员了”。

而是：

**我第一次觉得，原来我也可以把自己的想法做成一个可使用的工具。**

## 如果你也是零基础，我建议你这样开始

如果你也想尝试 Vibe Coding，不要一开始就做太大的项目。

可以从下面这些小工具开始：

- 高考分数线查询网站
- 简历生成器
- 番茄钟
- 学习计划表
- 资料整理工具
- 关键词选题工具

### 选题最好满足 3 个条件

1. 你自己真的需要
2. 功能能一句话说清楚
3. 第一版可以非常简单

比如我这个项目，第一版目标就是：

**输入条件，查到录取分数线。**

这样就足够开始了。

## 这个项目后面还能继续做什么？

目前它只是一个最小可用版本，后面还可以继续扩展。

### 我下一步想继续加的功能

1. 增加更多省份和学校数据
2. 增加按位次筛选
3. 增加“冲 / 稳 / 保”简单推荐
4. 增加学校详情页
5. 增加专业详情页
6. 增加数据来源说明
7. 最后部署成一个可以分享的网址

但因为这是高考相关工具，所以有一件事非常重要：

**数据一定要严谨。**

因此页面里也必须明确提示：

> 本工具仅供学习和志愿填报参考，最终请以各省教育考试院和高校官方数据为准。

## 最后想说的话

如果你也是文科生、零基础、甚至有点害怕代码，我真的建议你试一次。

不要先问自己：

**我会不会编程？**

而是先问自己：

**我想解决一个什么问题？**

然后把这个问题讲给 AI 听。

你会发现，做网站这件事，可能没有想象中那么遥远。
